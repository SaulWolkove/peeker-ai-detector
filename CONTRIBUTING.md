# Contributing to Peeker

Thank you for your interest in contributing to Peeker! This document provides guidelines and instructions for contributing.

## Getting Started

1. Fork the repository
2. Clone your fork:
   ```bash
   git clone https://github.com/YOUR_USERNAME/peeker-ai-detector.git
   ```
3. Install dependencies:
   ```bash
   npm install
   ```
4. Create a branch for your changes:
   ```bash
   git checkout -b feature/your-feature-name
   ```

## Development Workflow

1. Make your changes
2. Test the extension locally by loading it in Chrome
3. Run the build to ensure it compiles:
   ```bash
   npm run build
   ```
4. Commit your changes with a clear message

## Pull Request Process

1. Update the README.md if your changes affect usage or installation
2. Ensure your code follows the existing style
3. Test your changes thoroughly in Chrome
4. Submit a pull request with a clear description of the changes

## Code Style

- Use clear, descriptive variable and function names
- Add comments for complex logic
- Keep functions focused and modular

## Reporting Bugs

When reporting bugs, please include:

- Browser version
- Steps to reproduce the issue
- Expected behavior
- Actual behavior
- Screenshots if applicable

## Feature Requests

Feature requests are welcome! Please provide:

- A clear description of the feature
- Why it would be useful
- Any implementation ideas you have

## Questions?

Feel free to open an issue for any questions about contributing.
